# Лабораторная работа № 2

**СТРУКТУРЫ ДАННЫХ И ОПТИМИЗАЦИЯ АЛГОРИТМОВ**

**Цель работы:** изучить умножение матриц с комплексными элементами, оценить вычислительную сложность и производительность (MFlops), сравнить наивный алгоритм, библиотечный BLAS и собственную оптимизированную реализацию.

**Студент:** Ибнеев Никита Дмитриевич 090301-ПОВа-о25

**Задание:** Перемножить две квадратные матрицы размера 1024×1024 с элементами типа double complex. Оценить сложность по формуле c = 2n³ и производительность p = c/t·10⁻⁶ MFlops. Выполнить три варианта: (1) по формуле линейной алгебры; (2) cblas_zgemm (BLAS); (3) оптимизированный алгоритм (не ниже 30% от варианта 2).

**Реализация:** проект `lab2/`, хранение column-major, генерация матриц (random/formula/file), сборка MinGW + OpenBLAS (или Intel MKL при установке oneAPI).

## Результаты (n = 1024, источник formula)

| Метод | t, с | MFlops | % от BLAS |
|-------|------|--------|-----------|
| Naive (-O0) | 66.38 | 32.35 | 0.34 |
| cblas_zgemm | 0.225 | 9531.62 | 100 |
| Blocked + OpenMP | 0.640 | 3357.56 | 35.23 |

c = 2·1024³ = 2 147 483 648 операций.

Проверка: |C_opt − C_mkl|_max = 0 на тесте n = 128 и полном размере.

## Вывод

Наивный алгоритм имеет сложность O(n³) и на n = 1024 без оптимизаций компилятора крайне медленен. BLAS (cblas_zgemm) даёт наибольшую производительность за счёт векторизации и кэш-оптимизаций. Собственный блочный алгоритм с OpenMP достигает более 35% производительности BLAS при корректном результате и удовлетворяет требованию задания (≥ 30%).
